# Web Content Analyzer Skill

这个技能用于分析网页内容，提取关键信息，生成摘要，并检测潜在的 SEO 问题。

## 使用场景

- 用户提供 URL，要求分析网页内容
- 需要提取网页的主要内容和关键信息
- 要求生成网页内容摘要
- 需要评估网页的 SEO 友好度
- 分析竞品网页的内容策略

## 执行步骤

### 1. 获取网页内容

使用 `web_scraper` 工具获取目标 URL 的完整内容：
```
调用 web_scraper 工具，参数：
- url: 目标 URL
- extract_links: true（需要分析链接时）
- max_content_length: 5000
```

**注意**：优先使用 `web_scraper` 而不是 `web_search`，因为 `web_scraper` 可以获取完整网页内容。

### 2. 分析内容结构

从获取的内容中识别：
- 标题和副标题
- 主要段落和核心观点
- 关键词和主题
- 外部链接和引用
- 图片和多媒体内容描述

### 3. 生成分析报告

**必须严格按照以下 Markdown 格式输出**（每个章节使用二级标题，列表使用标准格式）：

```markdown
## 📊 网页分析报告

### 基本信息

- **页面标题**: [提取的标题]
- **主要内容类型**: [文章/产品页/ Landing Page/博客等]
- **预估字数**: [字数统计]

### 内容摘要

[200-300 字的内容摘要，包含核心观点]

### 关键信息提取

- **主要主题**: 
  1. [主题 1]
  2. [主题 2]
  3. [主题 3]

- **关键词**: [关键词 1]、[关键词 2]、[关键词 3]（5-10 个）

- **目标受众**: [推断的目标用户群体]

### SEO 分析

- **标题优化**: ⭐⭐⭐⭐⭐ [评估标题是否包含关键词]
- **内容质量**: ⭐⭐⭐⭐ [内容深度和原创性评估]
- **结构合理性**: ⭐⭐⭐ [H1/H2/H3 使用是否合理]

**改进建议**:
- [具体建议 1]
- [具体建议 2]
- [具体建议 3]

### 内容策略洞察

- **内容亮点**: [值得学习的地方]
- **改进空间**: [可以优化的方面]
- **竞品对比建议**: [如何差异化]
```

**格式要求**：
1. ✅ 每个章节必须使用 `###` 三级标题
2. ✅ 列表必须使用 `-` 或 `1. 2. 3.` 标准格式
3. ✅ 评分使用 ⭐ 符号（1-5 星）
4. ✅ 加粗文本使用 `**文本**`
5. ✅ 段落之间必须有空行
6. ❌ 禁止使用纯文本堆砌

## 注意事项

1. **内容准确性**: 基于实际获取的内容进行分析，不要臆测
2. **客观评价**: 保持中立的分析态度
3. **实用建议**: 提供具体可操作的改进建议
4. **隐私保护**: 不要存储或传播敏感的网页内容
5. **版权意识**: 提醒用户注意内容版权问题

## 常见错误处理

- **无法访问网页**: 告知用户并建议检查 URL 或网络
- **内容过少**: 说明页面内容不足，无法进行深入分析
- **动态加载内容**: 说明可能无法获取 JavaScript 渲染的内容
- **需要登录**: 告知用户无法访问需要认证的内容

## 示例

**用户输入**:
```
请帮我分析这个网页: https://example.com/blog/ai-trends
```

**执行流程**:
1. 调用 web_scraper 工具获取网页内容
2. 分析内容结构和关键信息
3. 生成完整的分析报告
4. 返回格式化的分析结果

**输出**: 见"生成分析报告"部分的标准格式
